En su forma m√°s fundamental, un programa se define como una **secuencia de declaraciones o sentencias dise√±adas para realizar una acci√≥n**. Incluso un script muy sencillo de una sola l√≠nea, como **`hola.py`**, cumple estrictamente con esta definici√≥n.

**La Resoluci√≥n de Problemas** El texto sugiere que la mejor manera de entender qu√© es un programa es pensar en un problema que necesita soluci√≥n.

- **El problema:** Se plantea el ejemplo de analizar grandes cantidades de texto para encontrar la palabra m√°s utilizada. Hacer esto manualmente ser√≠a lento y propenso a errores humanos.
- **La soluci√≥n:** Escribir un programa en Python permite realizar esta tarea con rapidez y precisi√≥n, liberando tiempo para actividades m√°s divertidas

### C√≥digo con palabras (`Contar_Palabras.py`):

```python
name = input('Enter file:')
handle = open(name, 'r')
counts = dict()

for line in handle:
    words = line.split()
    for word in words:
        counts[word] = counts.get(word, 0) + 1

bigcount = None
bigword = None
for word, count in list(counts.items()):
    if bigcount is None or count > bigcount:
        bigword = word
        bigcount = count

print(bigword, bigcount)
```

# 1.9. Los bloques de construcci√≥n de los programas

**Los seis bloques fundamentales** que componen la programaci√≥n:

1. **`Entrada (Input):`** Consiste en obtener datos del "mundo exterior". Esto puede ser leer datos de un archivo, recibir informaci√≥n de sensores (como un micr√≥fono o GPS), o, en los primeros programas que escribir√°s, recibir datos introducidos por el usuario a trav√©s del teclado.
    
    
2. **`Salida (Output):`** Es el proceso de mostrar los resultados del programa. Esto puede implicar mostrarlos en una pantalla, guardarlos en un archivo, o enviarlos a otros dispositivos como altavoces.
    
    
3. **`Ejecuci√≥n secuencial:`** Se refiere a ejecutar una sentencia (instrucci√≥n) tras otra, siguiendo estrictamente el orden en el que aparecen escritas en el script.
    
    
4. **`Ejecuci√≥n condicional:`** Permite comprobar ciertas condiciones y, bas√°ndose en el resultado, ejecutar o saltar (omitir) una secuencia de sentencias.
    
    
5. **`Ejecuci√≥n repetida:`** Consiste en ejecutar un conjunto de sentencias varias veces, generalmente con alguna variaci√≥n en cada repetici√≥n (lo que com√∫nmente llamamos bucles).
    
    
6. **`Reutilizaci√≥n:`** Implica escribir un conjunto de instrucciones una sola vez, asignarles un nombre y utilizarlas posteriormente en cualquier parte del programa cuando sean necesarias.
