## **2.10 Petición de información al usuario**

Python proporciona una función interna llamada **`input()`** para recibir datos desde el teclado. El programa se detiene y espera a que el usuario escriba algo y pulse Intro. 

>👨🏻‍🏫  
>**Todo lo que captura input() se devuelve como una cadena (str)**.


Puedes pasarle un texto a input() para que sirva de mensaje (prompt):

```python
>>> nombre = input('¿Cómo te llamas?\n')
¿Cómo te llamas?
Chuck
```

>👨🏻‍🏫  
>**`\n`** representa un **`newline`**, provocando un salto de línea.

</aside>

Si esperas que el usuario escriba un número, debes **`convertir`** la cadena usando **`int()`** o **`float()`**. Si el usuario escribe texto en lugar de números y tratas de usar **`int()`**, Python lanzará un error tipo **`ValueError`**

## **2.11 Comentarios**

A medida que los programas crecen, se vuelven difíciles de leer. Los **`comentarios`** comienzan con el símbolo **`#`** y todo lo que le sigue en esa línea es ignorado por Python.

Son vitales para explicar **por qué** el programa hace algo (la lógica humana), no **qué** hace (el código ya dice el qué).

## **2.12 Nombres de variables mnemónicos**

La palabra mnemónico significa "que ayuda a memorizar". Se recomienda usar nombres descriptivos por ejemplos horas y tarifa en lugar de a y b.

Aunque para Python ambos casos son exactamente idénticos, para el programador humano un buen nombre refleja el objetivo del programa.

## **2.13 Depuración (Debugging)**

Tipos de errores más comunes en este punto:

1. **`Errores de sintaxis:`** Usar nombres inválidos o palabras reservadas. Lanza un **`SyntaxError`**.
2. **`Errores en tiempo de ejecución (Runtime):`** Usar una variable antes de definirla. Lanza un NameError. Recuerda que Python es sensible a mayúsculas.
3. **`Errores semánticos:`** El programa corre sin fallos de Python, pero el resultado es incorrecto. Causado frecuentemente por mal orden en las operaciones.
