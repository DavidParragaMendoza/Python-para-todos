# **4.10 Funciones productivas y funciones estériles**

- **`Funciones productivas`**: Son aquellas que producen un resultado (devuelven un valor). Generalmente, querrás asignar ese resultado a una variable o usarlo en una expresión. Para devolver un valor desde dentro de una función, usamos la sentencia **`return`**.
    
    ```python
    def sumados(a, b):
        suma = a + b
        return suma
    
    x = sumados(3, 5)
    ```
    

- **`Funciones estériles (void functions)`**: Realizan una acción (como imprimir un mensaje en pantalla), pero no devuelven ningún valor matemático o lógico. Si intentas asignar el resultado de una función estéril a una variable, Python te devolverá un valor especial llamado **`None`**.

# **4.11 ¿Por qué funciones?**

Dividir un programa en funciones tiene múltiples beneficios:

1. **`Lectura y organización:`** Te permite dar un nombre descriptivo a un bloque de código, facilitando su comprensión.
2. **`Menos código repetido:`** Hacen el programa más pequeño. Si necesitas hacer un cambio futuro, solo lo corriges en un único lugar.
3. **`Depuración aislada:`** Te permite probar partes del programa de forma independiente.
4. **`Reutilización:`** Una función bien diseñada puede usarse en otros programas, ahorrándote tiempo.

# **4.12 Depuración**

Al escribir scripts (especialmente funciones), los problemas con los espacios en blanco y tabulaciones son comunes.

- **`Indentación:`** Python requiere consistencia. El mejor modo de evitar errores es **usar espacios exclusivamente** (típicamente 4 espacios) y no tabulaciones. Un buen editor de texto gestionará esto automáticamente.
- **`Asegúrate de ejecutar el archivo correcto:`** Un error de novato es corregir código en el editor, no guardarlo, y ejecutar una versión vieja repetidamente preguntándose por qué no se arregla. Si dudas, coloca un **`print("hola")`** al inicio; si no lo ves, estás ejecutando el archivo equivocado.
