# **4.7 Definición y usos**

Las sentencias dentro de una función no se ejecutan simplemente por definir la función. El resultado de definirlas es únicamente la creación del objeto función. **`Las sentencias solo se ejecutan cuando la función es llamada`**.

Por esto, **`la definición de la función debe ser ejecutada antes de que la función se llame por primera vez`**. Si intentas llamar a una función y la defines en una línea de código inferior, el programa arrojará un error.

# **4.8 Flujo de ejecución**

El **`flujo de ejecución`** es el orden en el que se ejecutan las sentencias.

1. La ejecución comienza en la primera sentencia del programa, de arriba hacia abajo.
2. Las definiciones de funciones (**`def`**) no alteran el flujo (Python toma nota de la función, pero salta su contenido).
3. Una **llamada a una función** actúa como un desvío: el flujo salta al cuerpo de la función, ejecuta sus sentencias, y luego vuelve exactamente al punto donde lo dejó.

Este proceso puede anidarse si una función llama a otra. Afortunadamente, Python siempre recuerda dónde estaba.

## **4.9 Parámetros y argumentos**

Muchas funciones necesitan información adicional para trabajar.

- **`Argumento`**: Es el valor real o variable que tú le pasas a la función al momento de llamarla (ej. **`math.sin(math.pi)`**).
- **`Parámetro`**: Es la variable que se usa **dentro** de la función para recibir y almacenar ese argumento.

```python
def muestra_dos_veces(bruce):
    print(bruce)
    print(bruce)

>>> muestra_dos_veces('Spam')
```

En el ejemplo, la palabra **`'Spam'`** es el argumento. Al ejecutarse la función, ese valor se asigna a la variable parámetro llamada `bruce`.

Si pasas una expresión o una variable externa como argumento, la expresión es evaluada primero, y el valor resultante es lo que llega al parámetro interno de la función. El nombre de tu variable externa no tiene nada que ver con el nombre del parámetro interno.
